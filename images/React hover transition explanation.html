In MongoDB, the $nin operator is used to match documents where the value of a field is not in a specified array. It's the opposite of the $in operator, which matches documents where the field's value is in a specified array.

Syntax:

js
{ field: { $nin: [value1, value2, ..., valueN] } }

Example 1: Find documents where a field's value is not in a list of values:

Assume you have a collection products with a field category, and you want to find all products that are not in the categories "Electronics" or "Clothing".

js
db.products.find({
    category: { $nin: ['Electronics', 'Clothing'] }
});

This query will return all documents where the category field does not contain either "Electronics" or "Clothing".

Example 2: Find documents where a field's value is not in a list of numbers:

If you have a users collection with a user_id field and you want to find users whose user_id is not in a given list:

js
db.users.find({
    user_id: { $nin: [1, 2, 3, 4] }
});

This will return all documents where the user_id is not one of 1, 2, 3, or 4.

Notes:

  • The $nin operator can be used with any data type, including strings, numbers, arrays, and even objects (though the comparison may depend on the field's data type).

  • If the field is missing or null, it will be considered as not matching the condition and will be included in the result set.

In MongoDB, the $and operator is used to combine multiple conditions and requires that all conditions must be true for a document to match. It's useful when you want to apply multiple conditions to the same document.

Syntax:

js
{ $and: [{ condition1 }, { condition2 }, ...] }

Each condition inside the $and array must be an individual query condition.

Example 1: Find documents that match multiple conditions

Assume you have a collection users with fields age and status. If you want to find users who are older than 25 and have a "active" status, you can use $and like this:

js
db.users.find({
    $and: [
        { age: { $gt: 25 } },
        { status: "active" }
    ]
});

This query will return all documents where:

  • age is greater than 25 ($gt: 25), and

  • status is equal to "active".

Example 2: $and with multiple conditions on the same field

You can also use $and to apply multiple conditions on the same field. For example, if you want to find documents where a price is greater than 10 and less than 100, you can do this:

js
db.products.find({
    $and: [
        { price: { $gt: 10 } },
        { price: { $lt: 100 } }
    ]
});

This will return products with a price between 10 and 100.

Example 3: $and without needing to explicitly use the operator

MongoDB also supports the implicit use of $and if you pass multiple conditions in a query. For example:

js
db.users.find({
    age: { $gt: 25 },
    status: "active"
});

This is equivalent to using $and, as MongoDB will automatically apply the $and logic when you pass multiple conditions at the same level.

Notes:

  • While you can use $and explicitly, in many cases, you can just combine multiple conditions directly (without $and) as shown in Example 3.

  • $and is especially useful when conditions need to be complex, or when you need to combine different types of operators across fields.

Yes, in MongoDB, you can use references between collections to model relationships, similar to foreign keys in relational databases. This is done using the ref keyword in the Mongoose schema to create a reference to another collection.

How it works:

In MongoDB, you can store the ObjectId of a document from one collection as a field in another collection. Using Mongoose, you can then use the ref field to tell Mongoose which model to look at for the referenced document, enabling you to populate the referenced fields.

Example:

Assume you have two collections: users and events, and you want to reference a user in the events collection.

1. Define the Schema for user and event:

js
const mongoose = require('mongoose');

// User schema
const userSchema = new mongoose.Schema({
    email: { type: String, required: true },
    name: { type: String, required: true }
});

// Event schema with reference to the User model
const eventSchema = new mongoose.Schema({
    title: { type: String, required: true },
    date: { type: Date, required: true },
    user: { type: mongoose.Schema.Types.ObjectId, ref: 'User' }, // Reference to the User model
});

// Create models
const User = mongoose.model('User', userSchema);
const Event = mongoose.model('Event', eventSchema);

2. Using populate to Get Data from the Referenced Collection:

When you query for events, you can use the .populate() method to replace the user field (which is just an ObjectId) with the full document from the User collection.

js
// Example: Find all events and populate the user field
Event.find()
    .populate('user')  // This will fetch the user data for each event
    .exec((err, events) => {
        if (err) {
            console.log(err);
        } else {
            console.log(events); // Each event will now include user data
        }
    });

Explanation:

  • In the eventSchema, the user field is defined as an ObjectId that references the User model using the ref keyword.

  • When you call .populate('user'), Mongoose will automatically replace the user field (which is an ObjectId) with the full User document for each event.

3. Example Query with .populate():

If you wanted to find all events where the user is John Doe:

js

    .then(events => {
        console.log(events); // This will include user data in the events
    })
    .catch(err => {
        console.log(err);
    });

What .populate() does:

  • Before populate(): The user field in the events collection would just contain the ObjectId (e.g., ObjectId("60c72b2f5f9b2c001f6b8b99")).

  • After populate(): The user field will be replaced with the full document of the user (e.g., { _id: ObjectId("60c72b2f5f9b2c001f6b8b99"), name: "John Doe", email: "john@example.com" }).

In JavaScript, you can use the following methods to see the keys of an object:

  1. Object.keys()
    This method returns an array of the object's own enumerable property names (keys).

    Example:

    javascript
    const obj = { name: "John", age: 30, city: "New York" };
const keys = Object.keys(obj);
console.log(keys); // ["name", "age", "city"]

  2. for...in loop
    You can also iterate over the object's keys using a for...in loop. This loop goes through all enumerable properties of the object.

    Example:

    javascript
    const obj = { name: "John", age: 30, city: "New York" };
for (let key in obj) {
    if (obj.hasOwnProperty(key)) {
        console.log(key); // name, age, city
    }
}

  3. Object.getOwnPropertyNames()
    This method returns an array of all the object's own property names, including non-enumerable properties.

    Example:

    javascript
    const obj = { name: "John", age: 30 };
const keys = Object.getOwnPropertyNames(obj);
console.log(keys); // ["name", "age"]

Which to use?

  • Object.keys(): Most common when you want just the enumerable property names.

  • for...in loop: Useful if you want to manually iterate through the keys.

  • Object.getOwnPropertyNames(): Use this when you need all properties, including non-enumerable ones.
